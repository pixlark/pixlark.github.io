Badge

03-05-2019

So I got pretty far with the last programming language, Winter, but eventually it got to be a bit of a mess. It's in a moderately functional state right now, but there are some unfortunate bugs with garbage collection that I don't have the patience to fix.

Over the last month or so, I've been working on a new langauge that I'm a lot more excited about. While it was really fun to finally make a real, working, moderately-sized language with Winter, it wasn't exactly my dream language. It was designed essentially just for ease-of-implementation. With my new language, Badge, I've actually put a lot of thought into the semantics of the langauge, and I think it puts together some really interesting ideas (all stolen from other languages, of course) in a cool way.

Here's one of the central ideas of the language: expressions are semantically "above" statements, not the other way around. In a language like C, the program is a series of statements, and expressions are just little pieces of those statements. In Badge (this is inspired by Scheme and Rust), expressions are the foremost object, and statements are all couched within expressions.

What does this mean when you're programming? It means that if you want to use a statement, like the let statement, you have to put it within a scope expression:

{
  let x = 12.
  println(x).
}.

The curly brackets denote a scope expression, and the period-terminated lines within are statements. Statements always have to be terminated with a period. If you write an expression and terminate with a period, it will take that expression as a statement and throw away the value it produces. However, if you don't period-terminate the expression, that returns from the surround scope expression, the whole thing evaluating to whatever you ended with. Here's an example:

println({
  let x = 12.
  set x = x + 2.
  x - 6
}).

This will print out 12 + 2 - 6, or 8.

The only place statements exist outside of scopes is at the very top-level of your program, which consists of a series of statements.

A traditionally statementful place that expressions take over in Badge are if-statements, or really if-expressions:

println(if 1 then 2 else 3).

This will print 2, because any value other than the unitary nothing value is treated as a true value. Of course, if you need to something stateful in an if-expression, you can just open a scope:

let x = 12.
if nothing then {
  set x = x + 1.
} else {
  set x = 0.
}.
println(x).

This will print out 0.
