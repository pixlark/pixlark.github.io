More on data structures and algorithms

04-05-2017

After a couple days of trying to teach myself about data structures, I learned two things:

  1. Data Structures and Algorithms are not seperate topics, in fact they are so intertwined that it's practically useless to try and learn them seperately.
  2. It's incredibly difficult to teach yourself about this topic just through wikipedia, because it's less about programming and more about math/logic.

So, I've decided to learn about this stuff through a course - specifically "Algorithms I" with Robert Sedgewick. Now, if you try and find this course you'll probably fail because Coursera removed it for some inexpicable reason. But, somebody on reddit has a backup of the whole course which you can probably find pretty easily through Google.

Standout songs: Hardwired, Atlas Rise!, and Spit out the Bone

Learning about data structures

27-04-2017

Started yet another repository today. Since I'm self-taught, I feel like I'm missing out on a lot of important formal topics. So, I'm using this repository to write implementations of some basic but important data structures:

  • Linked Lists
  • Queues
  • Stacks
  • Binary Trees
  • Hash Tables

And after that, I'll move onto algorithms, a topic that I know a little more about, but am still quite the novice:

  • Quicksort
  • Insertion sort
  • DFT
  • BFT
  • Heapsort
  • A*

Thanks to guthran for the lists.

Ludum Dare 38

25-04-2017

I participated in the Ludum Dare this year. For those unaware, it's a challenge where thousands of people try to create a fully-functional video game in 48 hours by themselves. You have to write your own code, draw your own art, foley your own sounds, and compose your own music.

My game, Planetary Defense, is probably the best game that I've made. Not code-wise - the code is awful - but it's the most playable and enjoyable thing I've made. I used the framework SFML, which is very new to me (the primary reason that my code is as bad as it is).

As someone who previously only used full engines like Unity, SFML is very interesting. I've been watching the YouTube series Handmade Hero, which is what convinced me to switch over to a framework. It's definitely more difficult to program in, but it's so freeing to have access to everything from the ground up. (Well, it handles the Windows API and OpenGL for me, but you know what I mean).

As a side note, the strange description to my game is meant to be a parody of those old Atari games where they would come up with ridiculously bad stories to try and legitimize the crappy graphics.

Album I can't stop listening to: Shaken By A Low Sound by Crooked Still
Standout songs: Little Sadie, Ain't No Grave, and Ecstasy
