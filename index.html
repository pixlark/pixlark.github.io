Ludum Dare 38

25-04-2017

I participated in the Ludum Dare this year. For those unaware, it's a challenge where thousands of people try to create a fully-functional video game in 48 hours by themselves. You have to write your own code, draw your own art, foley your own sounds, and compose your own music.

My game, Planetary Defense, is probably the best game that I've made. Not code-wise - the code is awful - but it's the most playable and enjoyable thing I've made. I used the framework SFML, which is very new to me (the primary reason that my code is as bad as it is).

As someone who previously only used full engines like Unity, SFML is very interesting. I've been watching the YouTube series Handmade Hero, which is what convinced me to switch over to a framework. It's definitely more difficult to program in, but it's so freeing to have access to everything from the ground up. (Well, it handles the Windows API and OpenGL for me, but you know what I mean).

As a side note, the strange description to my game is meant to be a parody of those old Atari games where they would come up with ridiculously bad stories to try and legitimize the crappy graphics.
